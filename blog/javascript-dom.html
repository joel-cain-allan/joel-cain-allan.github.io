Javascript Fundamentals

Let's get some of the basics down

2 April 2024

What is JavaScript?

At it's core, JavaScript is a (and the name gives this away) Scripting language. It is ubiquitous on the client-side of the web, and according to wikipedia present in 99% of websites.

At the most basic level, JavaScript is a way to tell webpages to display content dynamically, rather than being purely the equivalent of a poster put on a wall. You can see this in action on this blog - the dog image in the header is randomly selected between two options, and Javascript is what allows that selection (and the subsequent display of the selected image). The HTML and CSS represent the static content of a web-page, and generally speaking are what you actually see, while JavaScript (JS) is what does the actual heavy lifting.
A much-used analogy is one of a house. The blueprint of the house's framing, wall locations etc are HTML. The decorating choices, paint colours, fixture design etc are CSS. The plumbing, wiring, heating and appliances are JavaScript.


Okay, but how do you actually write it?

That's a very expansive question! Let's focus on some of the core concepts.

You can break down the logic behind JavaScript (or any object-oriented language) into some smaller parts:

  • Objects
  • Functions
  • Control Flow

Objects

Objects are a way of representing information in a container. That sounds vague and tricky, but let me give you an example: 

let favouriteColor = 'black'

That piece of JavaScript represents declaring a variable (a type of object), with the let keyword, naming that variable favouriteColour, and giving it a value of "black"

You could now call that favouriteColour variable elsewhere in your code, and retrieve that "black" value. That might not sound useful just yet, but it's one of the most core features of any programming language.



Objects go far beyond just simple single-layer variable declarations however, they can also represent collections.
Consider the scenario where we have a group of people, and we want to record all of their favourite colours. Declaring a variable for each like


let tomsColour = 'blue'
let gregsColour = 'white'
let sarahsColour= 'green'

would be slow, tedious, and introduce a whole lot of variables into the codebase. We could, instead, declare a more complex object to hold those values 


let favouriteColours = {}

And then can add, change, or retrieve values from this new object with a key, in this case the person's name. 


favouriteColours.greg = 'white'
favouriteColours.sarah = 'blue'
let color = favouriteColours.sarah 
//color is now equal to blue!

Try out setting a variable for yourself - the blue text below is equal to whatever you write in the little textbox. Change it to whatever you like!


Arrays

An Array is a type of object that is a collection, or represents a group of multiple other objects. These elements don't have an easy way to reach them with a key like the above example. Why would you want to use a collection like this? Well, consider your sock drawer. Your socks probably don't have names, or other ways you refer to them, but you do have a collection of socks in there. In code, that sock drawer array might look like this: 


let sockDrawer = ['sockPair1', 'sockPair2', 'looseSock']

Unlike objects, you can't fetch values with a key - sockDrawer.sockPair2 is not going to achieve anything. Instead, for arrays, you fetch their values with an index - just the number representing which element they are in the array. However, there's something to keep in mind with this... 


let socks = sockDrawer[1]
//socks is equal to the value 'sockPair2'!

In Javascript, like most programming languages (I'm looking at you Lua!), we begin counting with 0. So the first index in that array is a 0, the second is 1, etc.


Functions

Functions are the meat and potatoes of code, and introduce reusable logic. They are an encapsulated set of instructions that can accept parameters (ie user-defined objects or functions) and use them as part of their logic. This is very significant - without functions we would need to re-declare logic constantly (and our control flow would be disgusting, but that's a future topic).

Consider a task like cooking a meal, there are many small parts of the recipe that must be performed, but many of those parts are actually basic, reusable skills that are used for all meals. Cutting vegetables, for instance, is always much the same process. 


function makeCaramelizedOnions(){
  cutVegetables(onions)
  //other cooking logic here
}

function makeCarrotSticks(){
  cutVegetables(carrots)
}

function cutVegetables(vegetable){
  //logic for cutting a vegetable lives here!
}

In a world without functions, we'd instead have to redefine (or copy-paste) the logic for cutting vegetables every time!

In addition to saving duplication, it lets us break up our code into smaller, more manageable and testable chunks.


Control Flow

The control flow is the order in which code is executed. At the most basic level, code executes from top-to-bottom.
For anything but the most basic script, however, there will be wrinkles in that process - there will be loops (running the same piece of code multiple times), function calls (running a piece of code defined elsewhere), and it gets even more tricky once you start bringing in multiple threads.
When you calling a function, the flow moves to that function, the code defined in that function is executed, and then the control flow returns to where it was before calling that function. But functions can also call other functions (or even themselves!), so sometimes the control flow isn't straight-forward to understand.

See this flow example: 


console.log("a script")

function helloFunction(){
  console.log('hello')
}

function goodbyeFunction(){
  console.log('goodbye')
}

function helloAndGoodbyeFunction(){
  helloFunction()
  goodbyeFunction()
}

helloFunction()
goodbyeFunction()
helloFunction()


Can you guess what the order of things in the console log might look like?
Here's the executed order: 


a script
hello
goodbye
hello
goodbye
hello

A consistent and function-driven flow is useful, but we've yet to cover a core feature - what if we want to execute some code more than once? Well, there's where loops come in!


Loops

A loop allows code to be executed multiple times sequentially, often with slightly different parameters. Let's look at a basic example of a loop in action: 


for(let i = 0; i<10; i++){
  console.log(i)
}

This loop sets a variable 'i' to the value 0 with let i = 0, tells the loop to run until i is equal to 10 with i < 10, and tells the loop to increase i by one after each loop with i++, (ensuring that the loop won't continue infinitely).

On each iteration of the loop the console logs the value of i, first logging 0, then 1, then 2 ... until finally logging 9.
Then, when i meets the termination condition of being greater than 9 the loop terminates, and the control flow continues on as before it entered the loop, in the same top-to-bottom manner


That's all well and good, but how does JS actually interact with the HTML and CSS?

This is where the DOM (Document Object Model) comes in.

The DOM is a document that consists of a logical tree, from a root down many branches that terminate in nodes. This DOM represents the entirety of the web-page, and ultimately a web-browser interprets this DOM to display a webpage to you.
This DOM isn't entirely set in stone - JavaScript can manipulate the DOM through a number of interfaces. For example, this page sets the dog face on the header by using the document interface: 


let headerImage = document.createElement('img')

if (getRandomInt(2) > 0) {
  headerImage.src = '/images/archie-head-big.png'
  headerImage.alt = 'apprehensive black and tan dog face'
} else {
  headerImage.src = '/images/cindy-head-big.png'
  headerImage.alt = 'cheeky fluffy yellow dog face'
}

headerImage.id = 'dog-head'
headerImage.width = '150'
headerImage.height = '150'

The conditional (if, else) might be confusing, but hopefully you can see that document.createElement is used to create an <img> HTML element named headerImage, and then that element's src, alt, id, width, and height properties are set.
Finally, that headerImage is added to an existing span in the DOM like so: 


headerContentSpan.appendChild(headerimage)
